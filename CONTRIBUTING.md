# Contributing to GitMind

GitMind is an open reasoning environment. Contributions are not just welcome — they are the mechanism by which this document evolves.

This document follows the same principles it describes: every contribution carries its reasoning, and changes are reviewed for semantic coherence, not just textual fit.

---

## How to Contribute

### 1. Fork the repository

Get a private copy of the document and its full history. Explore it freely. Use an LLM to navigate it, ask questions, find gaps.

### 2. Add your idea

Create a new file in `ideas/`:

```
ideas/your-concept-name.md
```

Use this template:

```markdown
# [Idea Title]

> **Proposed placement:** Section X or new section
> **Status:** Draft / Ready for review
> **Author:** your name or handle

---

[Your idea. Include: what it is, why it matters, how it connects to existing content, open questions.]
```

### 3. Or propose a change to an existing section

Edit the relevant file in `sections/`. In your PR description, explain:
- What you changed
- Why (what reasoning led you here)
- What alternatives you considered and rejected

The *why* is the contribution. The text change is just its surface.

### 4. Open a Pull Request

The owner reviews not just the text but the reasoning behind it. Be prepared to discuss:
- Where your thinking diverges from the current document
- What evidence or argument supports your proposed change
- Whether the change achieves semantic coherence with adjacent sections

---

## What makes a good contribution

- **Precise** — proposes a change to a specific place for a specific reason
- **Reasoned** — the PR description contains the argument, not just the conclusion
- **Coherent** — the change integrates with the existing reasoning without creating contradictions
- **Honest about uncertainty** — open questions are valuable; don't paper over them

---

## What this is not

- A comment thread. If you have a reaction, turn it into a reasoned proposal.
- A place to rewrite everything. Contribute precisely where you have something to add.
- A place to add conclusions. Contribute the reasoning that leads to them.

---

## Quick navigation for new contributors

1. Start with `CONTEXT.md` — the 2-minute LLM brief
2. Read `INDEX.md` — the full section map
3. Find where your thinking connects
4. Add to `ideas/` or propose a section change
