# 2. Problems We Need to Address

> **One-line summary:** Three core problems block knowledge collaboration: synthesis, dimensionality, and merge.
> **Navigation:** [INDEX.md](../INDEX.md) | Prev: [01_introduction.md](01_introduction.md) | Next: [03_solution.md](03_solution.md)

---

## Problem 2.1 — The Synthesis Problem & The Transition Tax

Before anyone can collaborate on knowledge the way developers collaborate on code, there needs to be a shared, structured artifact that carries reasoning with it. Code has syntax. It has structure. A programming language is a shared protocol that makes code readable, executable, and mergeable by anyone who knows the language.

Knowledge produced in conversation has none of that structure yet. It's fluid, contextual, and locked inside a chat window — not because the thinking isn't valuable, but because moving it into a shareable, structured format carries a prohibitively high **Transition Tax**. The cost in effort and fidelity loss is high enough that the loop rarely closes. Thinkers don't synthesize their conversations into documents. Documents don't feed back into thinking. The cycle between *thinking* and *documenting* is broken.

There is no functioning loop. You cannot think a document into existence. Before collaboration is possible, the transition tax must drop to near zero — so that the act of thinking and the act of creating a shareable artifact become the same act.

---

## Problem 2.2 — The Dimensionality Problem

Even with a structured artifact, current collaboration tools only expose the surface. A document is a flat surface. You read it from top to bottom. Unlike a GitHub repository — where you navigate into the specific file, function, or commit that is relevant to you — a document forces a linear, passive consumption of its conclusions. You can comment on the text. But you cannot enter the reasoning behind it. You are a spectator to a thought process that is already over. The reasoning is invisible, and with it, the ability to contribute at depth.

---

## Problem 2.3 — The Merge Problem

Even with structured artifacts and multidimensional access, collaboration requires a mechanism for reconciling two reasoning chains without losing what either side discovered. In code this is the pull request and the merge. In knowledge it has never existed.
